  + 领域设计：获取特定领域软件架构（DSSA）；
  + 领域实现：组合可重用分析；
+ 参与人员
  + 领域专家：提供关于领域中系统的需求规范和实现的知识，帮助组织规范的、一致的领域字典，帮助选择样本系统作为领域工程的依据，复审领域模型；
  + 领域分析师：
  + 领域设计人员：根据领域模型和现有系统开发出DSSA并验证，建立联系；
  + 领域实现人员：根据领域模型和DSSA，或者从头开发可重用构件，利用再工程技术从现有系统中提取可重用构件，对可重用构件进行验证，建立联系。

#### C/S和B/S
+ C/S：客户端-服务器
  + 优点：
    + 个性化：客户端独立设计；
    + 响应速度快：充分发挥客户端PC的处理能力；
    + 安全性能保障：客户端是独立开发的；
  + 缺点：
    + 维护成本高：客户端是不需要重复安装，所以用户可以不更新与升级；
    + 开发成本高：需要开发客户端和服务器两套程序；
+ B/S：浏览器-服务器
  + 优点：
    + 维护成本低：不需要安装客户端，只需要更新后台代码即可实现所有客户端的更新；
    + 开发成本低；
    + Web交互性强：
  + 缺点：
    + 耗流量，每次都要加载内容；
+ 区别：
  + 硬件环境不同
  + 安全要求不同
  + 程序架构不同
  + 软件重用不同
  + 系统维护不同
  + 处理问题不同
  + 用户接口不同
  + 信息流不同
+ C/S与B/S混合技术结构
  + 纽带是数据服务器，一方面数据服务器要响应应用服务器的请求，另一方面要同时响应web服务器的请求。在此结构体中，c/s结构部分主要解决纯b/s系统对用户请求响应速度慢的问题，而b/s结构部分则打破了用户群只在局域网中的困局。

#### 系统质量
+ 可用性
  + 实现：心跳，ping/echo，主动冗余，被动冗余，选举；
  + 举例：断电恢复；
+ 性能
  + 实现：增加计算资源，减少计算开销，引入并发机制，采用资源调度；
  + 举例：1000人请求0.5s响应；
+ 可修改性
  + 实现：接口-实现分离，抽象，信息隐藏；
+ 安全性
  + 实现：追踪审计；
  + 举例：对机器人的远程控制命令进行加密，抵挡恶意入侵破坏行为，并对攻击进行报警和记录。

#### 性能指标
+ 计算机：时钟频率（主频）、运算速度/精度、内存容量、存取周期、数据处理速率（PDR）、吞吐量、响应时间、利用率、平均故障响应时间、兼容性、可扩充性、性价比；
  + RASIS
    + 可靠性reliability
    + 可用性availability
    + 可维护性
    + 完整性
    + 安全性
+ 网络
  + 网络设备：吞吐量、延迟、丢包率、转发速度；
  + 网络：可达性、吞吐量、传输速度、信道利用率/容量、带宽利用率、丢包率、平均传输延迟、平均延迟抖动；
  + 应用：QoS、网络对语言/视频应用支持程度；
+ 操作系统：系统可靠性、吞吐量、响应时间、资源利用率、可移植性；
+ 数据库：最大连接数、允许索引数量、最大并发实物处理能力、负载均衡能力；
